    // why qmd-rs

    Built for speed. Built for humans and agents.

    Everything you need to search your notes locally — without sacrificing power or AI compatibility.

    Rust-native performance

    Zero JVM, no Python, no ML runtime baggage. A lean single binary with sub-20ms cold start.

    SQLite + FTS5 core

    All data stays local in a single SQLite file. Full-text search via FTS5 with vector-ready chunk storage.

    Hybrid search pipeline

    Parallel BM25 + vector retrieval fused with Reciprocal Rank Fusion (k=60) and a reranker pass.

    OpenAI-compatible APIs

    Plug in any OpenAI-compatible endpoint — Ollama, llama.cpp, or cloud LLMs for embeddings & reranking.

    MCP agent surface

    Six MCP tools exposed over stdio or HTTP/SSE. Drop qmd-rs into any AI agent workflow instantly.

    Smart markdown chunking

    Semantic boundary detection with heading breaks, code fence protection, and overlap for better recall.

    llm= "llama3.2:3b"

    // model context protocol

    First-class AI agent support.

    Six MCP tools ready to drop into any agent workflow — via stdin/stdout JSON or HTTP/SSE.

    AVAILABLE TOOLS

    qmd_search

    BM25 full-text search over indexed markdown chunks

    qmd_vector_search

    Semantic vector similarity search using embeddings

    qmd_deep_search

    Full hybrid pipeline: expand → BM25+vec → RRF → rerank

    qmd_get

    Retrieve a single document by ID or path

    qmd_multi_get

    Retrieve multiple documents via glob pattern

    qmd_status

    Return engine health, document count, API status

    stdio

    Full toolset via JSON lines on stdin. Perfect for local agent runners.

    qmd mcp
    HTTP/SSE

    SSE heartbeat stream + tool POST endpoint for remote agents.

    qmd mcp --http

    HTTP ENDPOINTS

    GET/events→ SSE heartbeat stream
    POST/tool→ full toolset (search, vsearch, deep_search, get, multi-get, status)

    // get started

    Up and running in minutes.

    01Install from crates.io
    $ cargo install qmd
    02Add a collection
    $ qmd collection add /path/to/notes
    03Index your markdown
    $ qmd embed
    04Search
    $ qmd search "rust error handling"
    $ qmd vsearch "safe async abstraction"
    $ qmd query "best chunking strategy"
